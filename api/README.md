# 接口文档总览

所有 HTTP API 默认以 `/api/v1` 为前缀，管理端接口以 `/api/v1/admin` 为前缀。统一响应、分页、错误码和字段命名见 [接口规范](./api-spec.md)。

## 已有接口文档

- [认证与用户](./auth-users.md)：用户注册登录、资料、头像、地址、积分、会员基础能力。
- [商品与店铺](./products-merchants.md)：分类、商品、SKU、图片、评价、收藏、店铺主页、店铺关注。
- [购物车、订单与支付](./cart-orders.md)：购物车、结算、跨店订单、订单状态、发货收货、评价、售后、支付单。
- [支付宝支付](./alipay-payment.md)：支付宝沙箱扫码支付预创建、同步、异步通知。
- [促销、积分与会员](./promotions.md)：优惠券、满减、积分抵扣、会员积分配置。
- [拼团](./group-buy.md)：商家拼团活动、用户开团、参团、成团。
- [社区](./community.md)：帖子、评论、点赞、分区、话题、用户主页、种草来源。
- [管理端](./admin.md)：平台端与商家端认证、商家入驻、商品、订单、售后、促销、社区治理。
- [上传](./upload.md)：本地图片上传。
- [WebSocket、客服与 AI 助手](./websocket.md)：客服会话、消息持久化、HTTP 兜底发送、WebSocket 推送和用户端 AI 购物助手。

## 重要约定

- 当前支付链路采用支付宝沙箱扫码支付。
- 平台端和商家端 session 分离，允许同时登录。
- 平台不创建店铺或商品；店铺来自商家入驻审核通过，商品由商家创建。
- 只有商家入驻需要事前审核；商品、帖子、评论、评价采用发布后治理。
- 商家端接口必须按当前 `merchant_id` 做后端权限校验，不能只靠前端过滤。
- 金额统一使用分，字段名以 `_cent` 结尾；前端展示为元。
- 页面必须展示关键业务信息和 ID，接口 JSON 返回只用于排查。

## 后续新增接口要求

新增接口前先补本文档目录和对应模块文档。接口文档至少包含：

- 路径和方法。
- 认证与权限。
- 查询参数或请求体。
- 响应字段。
- 状态枚举。
- 错误码。
- 分页规则。
- 对前端页面的展示要求。
